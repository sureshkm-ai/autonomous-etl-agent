DATA FLOW SUMMARY
1. User POSTs UserStory JSON
2. DB persists story + creates run
3. Audit: STORY_SUBMITTED + RUN_CREATED
4. Background: LangGraph streams nodes
5. Each LLM call → prompt hash + token track
6. After tests: approval_gate check
• confidential/restricted → HOLD
• budget >75% → HOLD
7. PRAgent → GitHub PR + Issue
8. DeployAgent → S3 (tagged) + Airflow
9. Audit: RUN_COMPLETED event
